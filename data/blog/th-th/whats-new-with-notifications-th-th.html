ย้อนไปเมื่อปี 2019 เรา เราเขียนบล็อกเกี่ยวกับศูนย์แจ้งเตือนใหม่ของเรา เพื่อเป็นศูนย์กลางในการสร้างการแจ้งเตือนบนบัญชีของคุณ ตั้งแต่นั้นเป็นต้นมาเราได้พูดเกี่ยวกับการแจ้งเตือนแบบใหม่ที่คุณสามารถติดตั้งได้ แต่ไม่ค่อยพูดเกี่ยวกับการอัปเดตแพลตฟอร์มการแจ้งเตือน ดังนั้น มีอะไรใหม่ในการแจ้งเตือน?

ทำไมเราถึงให้ความสนใจกับการแจ้งเตือน

เรารู้ว่าการแจ้งเตือนเป็นสิ่งที่สำคัญมากต่อลูกค้าของเรา Cloudflare อยู่ตรงกลางระหว่างทรัพย์สินทาง Internet ของคุณกับคนทั้งโลก เมื่อมีบางอย่างผิดปกติ คุณต้องการรู้ในทันทีเพราะมันอาจส่งผลกระทบครั้งใหญ่ต่อผู้ใช้ของคุณได้ แต่คุณไม่อยากนั่งเฝ้า แดชบอร์ด Cloudflare ทั้งวัน กดรีเฟรชหน้าการวิเคราะห์ซ้ำๆ เพื่อให้มั่นใจว่าคุณจะไม่พลาดสิ่งสำคัญใดๆ การแจ้งเตือนจึงจะเข้ามามีส่วนช่วยในจุดนี้ แทนที่จะต้องนั่งเฝ้าทรัพย์สินทาง Internet ของคุณ คุณต้องการให้ Cloudflare แจ้งคุณโดยตรงเมื่อมีบางอย่างผิดปกติเกิดขึ้น

Cloudflare มีการแจ้งเตือนหลายประเภทเพื่อให้คุณมั่นใจได้ว่าคุณจะไม่พลาดข้อมูลสำคัญใดๆ เรามีการแจ้งเตือนที่จะบอกคุณว่า คุณถูกปฏิเสธการให้บริการแบบกระจาย หรือว่า ไฟร์วอลล์บล็อกการร้องขอมากกว่าปกติ หรือว่า พบข้อผิดพลาดข้อมูลต้นทางระดับสูงกว่า 5xx ข้อผิดพลาด หรือแม้แต่ว่า การใช้งาน CPU ของสคริปต์ Workers สูงกว่าปกติ เราได้เพิ่มการแจ้งเตือนอย่างต่อเนื่อง ดังนั้นอย่าลืมตรวจสอบ เอกสารการพัฒนา Cloudflare ของเราเพื่อดูว่ามีอะไรใหม่!

หมดยุคของอีเมลแล้ว สมัยนี้ต้อง Webhook

เรามีการแจ้งเตือนที่ยอดเยี่ยมนี้แล้ว แต่ วิธีที่จะแจ้งเกี่ยวกับกิจกรรมนั้นเป็นอย่างไร? คำตอบสุดคลาสสิกคงเป็น "เราส่งอีเมลถึงคุณ" ลูกค้าของเราสามารถกำหนดการแจ้งเตือนให้ส่งไปที่อีเมลที่ต้องการได้

อย่างไรก็ตาม อีเมลนั้นไม่ใช้ตัวเลือกที่ดีที่สุด จะเกิดอะไรขึ้นหากอีเมลถูกส่งไปที่สแปม หรือถูกจัดไปอยู่ในโฟลเดอร์อื่นที่คุณไม่ได้เปิดดูบ่อยๆ จะเป็นอย่างไรหากคุณคือคนที่ไม่เคยเคลียร์กล่องข้อความขาเข้าเลย และมีอีเมลที่ยังไม่ได้อ่านอยู่กว่าสี่พันอีเมลที่อาจทำให้อีเมลสำคัญใหม่ๆ ที่ส่งเข้ามาคลาดสายตาไป คุณต้องการวิธีที่การแจ้งเตือนจะส่งตรงไปยังแพลตฟอร์มการส่งข้อความที่คุณเปิดดูมากที่สุด ไม่ว่าจะเป็น Slack หรือ Microsoft Teams หรือ Discord หรืออย่างอื่นทั้งหมด สำหรับลูกค้าแผนมืออาชีพ ธุรกิจ และองค์กร Webhook จะเข้ามาช่วยในจุดนี้

Webhook นั้นทรงพลังอย่างไม่น่าเชื่อ! Webhook เป็น API ประเภทหนึ่งที่มีพฤติกรรมที่เรียบง่ายและมีมาตรฐาน ซึ่งจะอนุญาตให้การบริการหนึ่ง (Cloudflare) ส่งข้อมูลกิจกรรมตรงไปยังการบริการอื่นได้ การบริการปลายทางสามารถเป็นอะไรก็ได้ ทั้งแพลตฟอร์มการส่งข้อความ ระบบการจัดการข้อมูล ระบบเวิร์กโฟลว์อัตโนมัติ หรือแม้แต่ API ภายในของคุณ

ในขณะที่ Cloudflare มีการรับรองที่ยอดเยี่ยมมาระยะหนึ่งแล้วสำหรับการใช้ Webhook ใน Slack, Microsoft Teams, Google Chat และ API ของลูกค้า เมื่อเร็วๆ นี้ เราได้เพิ่มการรองรับ DataDog, Discord, OpsGenie และ Splunk อีกด้วย คุณสามารถอ่านข้อมูลเพิ่มเติมสำหรับวิธีการติดตั้ง Webhook ประเภทต่างๆ ได้ที่ เอกสารการพัฒนา Cloudflare

เพราะ Webhook นั้นใช้งานได้หลากหลาย และยิ่งมีผู้ใช้งานมากขึ้นเรื่อยๆ จำนวน Webhook กำหนดค่าไว้ในระบบการแจ้งเตือนของ Cloudflare เป็นสองเท่าของค่าปกติ ทุกๆ สามเดือน ลูกค้าสามารถสร้าง Webhook ได้ในแท็บ การแจ้งเตือนในแดชบอร์ด

คนที่ลืมประวัติศาสตร์ถูกกำหนดให้ทำมันซ้ำอีก

Webhook นั้นเจ๋งมาก แต่ก็ยังมีข้อผิดพลาดอยู่ จะเกิดอะไรขึ้นหากคุณได้รับการแจ้งเตือนและเผลอลบมันไป หรือเมื่อมีพนักงานใหม่ในบริษัทคุณ แต่คุณลืมอัปเดตการตั้งค่าการแจ้งเตือนให้ส่งไปให้พนักงานใหม่ด้วย

ก่อนหน้านี้ การแจ้งเตือนของ Cloudflare ทั้งหมดจะเกิดขึ้นในเวลาหนึ่ง เราส่งการแจ้งเตือนด้วยวิธีการที่คุณต้องการและเราก็จะไม่เห็นการแจ้งเตือนนั้นอีกต่อไป ถ้าการแจ้งเตือนนั้นหายไปจากปลายทาง เราไม่มีทางที่จะช่วยกู้ข้อมูลที่อยู่ในนั้นคืนมาได้เลย

ประวัติการแจ้งเตือนจึงเข้ามาแก้ปัญหานี้โดยเฉพาะ ผู้ใช้สามารถดูรายการบันทึกการแจ้งเตือนที่ถูกส่งไปได้ เวลาที่ส่ง และใครเป็นคนส่ง ลูกค้าที่ใช้งานแผนฟรี มืออาชีพ หรือธุรกิจ จะสามารถดูประวัติการแจ้งเตือนได้เป็นเวลา 30 วัน ลูกค้าแผนองค์กรจะสามารถดูประวัติการแจ้งเตือนได้ 90 วัน

ตอนนี้ ประวัติการแจ้งเตือนสามารถ ใช้งานได้ผ่าน API เท่านั้น แต่โปรดติดตามการอัปเดตโดยตรงที่ แดชบอร์ด Cloudflare!
