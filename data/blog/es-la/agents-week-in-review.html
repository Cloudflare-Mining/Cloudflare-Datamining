Hoy finaliza nuestra primera Agents Week, una semana de la innovación dedicada por completo a la era de los agentes. No podría haber sido más oportuno: durante el último año, los agentes han cambiado rápidamente la forma de trabajar de las personas. Los agentes de codificación ayudan a los desarrolladores a realizar entregas más rápido que nunca. Los agentes de soporte resuelven los tickets de un extremo a otro. Los agentes de investigación validan las hipótesis de cientos de fuentes en cuestión de minutos. Y las personas no solo ejecutan un agente, sino que ejecutan varios en paralelo, las 24 horas del día.

Como señalaron el director técnico de Cloudflare, Dane Knecht, y la vicepresidenta de producto, Rita Kozlov, en nuestra publicación de bienvenida a Agents Week, la escala potencial de los agentes es asombrosa: si solo una fracción de los trabajadores del conocimiento del mundo ejecuta cada uno unos pocos agentes en paralelo, necesitas capacidad informática para decenas de millones de sesiones simultáneas. El modelo de "una aplicación para muchos usuarios" en el que se construyó la nube no funciona para eso. Pero eso es exactamente lo que los desarrolladores y las empresas buscan: crear agentes, ponerlos a disposición de los usuarios y ejecutarlos a gran escala.

Llegar allí significa resolver problemas en toda la pila. Los agentes necesitan una capacidad de cómputo que escale desde sistemas operativos completos hasta aislamientos ligeros. Necesitan seguridad e identidad integradas en cómo funcionan.  Necesitan una caja de herramientas para los agentes: los modelos, las herramientas y el contexto adecuados para hacer un trabajo real. Todo el código que generan los agentes necesita una ruta clara desde el prototipo inicial hasta la aplicación de producción. Y, por último, a medida que los agentes generan una proporción cada vez mayor del tráfico de Internet, la propia web debe adaptarse a la emergente web de agentes. Resulta que la plataforma informática sin contenedores y sin servidor que lanzamos hace ocho años con Workers estaba preparada para este momento. Desde entonces, la hemos convertido en una plataforma completa, y esta semana lanzamos una nueva serie de primitivas diseñadas a medida para agentes, organizadas precisamente en torno a esos problemas.

Estamos aquí para crear Cloud 2.0, la nube de agentes. Infraestructura diseñada para un mundo en el que los agentes son una carga de trabajo principal. 

Aquí tienes una lista de todo lo que hemos anunciado esta semana, ¡no te pierdas nada!

Proceso

Comienza con la computación. Los agentes necesitan un lugar donde ejecutarse, y un lugar donde almacenar y ejecutar el código que escriben. No todos los agentes necesitan lo mismo: algunos necesitan un sistema operativo completo para instalar paquetes y ejecutar comandos de terminal, la mayoría necesita algo ligero que se inicie en milisegundos y escale a millones. Esta semana lanzamos los entornos para ejecutarlos, así como un nuevo espacio de trabajo compatible con Git para los agentes:

Resumen

Artifacts: almacenamiento con control de versiones integrado con Git

Ofrece a tus agentes, desarrolladores y automatizaciones un espacio para guardar su código y sus datos. Acabamos de lanzar Artifacts: almacenamiento con control de versiones integrado con Git diseñado para agentes. Crea decenas de millones de repositorios, bifúrcalos desde cualquier servidor remoto y usa una URL en cualquier cliente Git.

Los agentes cuentan con sus propias computadoras dentro de Sandboxes en disponibilidad general

Los Sandboxes de Cloudflare ofrecen a los agentes de IA un entorno persistente y aislado: una computadora real con acceso a shell, un sistema de archivos y procesos en segundo plano, que se inicia a pedido y retoma exactamente donde se había dejado.

Dinámicos, seguros y con reconocimiento de identidad: controles de salida para Sandboxes

Outbound Workers para Sandboxes ofrecen un proxy de salida programable y Zero Trust para agentes de IA. Esto permite a los desarrolladores inyectar credenciales y aplicar políticas de seguridad dinámicas sin exponer tokens confidenciales a código no fiable.

Durable Objects en Dynamic Workers: Dale a cada aplicación generada por IA su propia base de datos

Durable Object Facets permite a Dynamic Workers crear instancias de Durable Objects con sus propias bases de datos SQLite aisladas. Esto permite a los desarrolladores crear plataformas que ejecutan código persistente y con estado generado sobre la marcha.

Reestructuración del plano de control de Workflows para la era agéntica

Cloudflare Workflows, un motor de ejecución persistente para aplicaciones de varios pasos, ahora 50.000 ejecuciones concurrentes y una tasa de creación de 300 por segundo gracias a un plano de control rediseñado, lo que permite el escalado para los casos de uso de agentes persistentes en segundo plano.

Seguridad

Ejecutar agentes y su código es solo la mitad del desafío. Los agentes se conectan a redes privadas, acceden a servicios internos y realizan acciones autónomas en nombre de los usuarios. Cuando cualquiera en una organización puede crear sus propios agentes, la seguridad no puede ser una ocurrencia tardía. Tiene que ser la opción por defecto. Esta semana, lanzamos las herramientas para facilitarlo.

Resumen

Redes privadas seguras para todos: usuarios, nodos, agentes, Workers: presentamos Cloudflare Mesh

Cloudflare Mesh brinda acceso a la red seguro y privado para usuarios, nodos y agentes de IA autónomos. Al integrarse con Workers VPC, los desarrolladores ahora pueden otorgar a los agentes acceso limitado a bases de datos privadas y API sin túneles manuales.

El sistema de OAuth gestionado para Access: prepara las aplicaciones internas para agentes con un solo clic

El sistema de OAuth gestionado para Cloudflare Access ayuda a que los agentes de IA naveguen de forma segura por aplicaciones internas. Al adoptar RFC 9728, los agentes pueden autenticarse en nombre de los usuarios sin recurrir a cuentas de servicio poco seguras.

Protección de identidades no humanas: revocación automatizada, OAuth y permisos delimitados

Cloudflare presenta tokens API escaneables, mayor visibilidad en OAuth y disponibilidad general (GA) de permisos delimitados por recurso. Estas herramientas ayudan a los desarrolladores a implementar una verdadera arquitectura de privilegios mínimos y al mismo tiempo protegen contra la filtración de credenciales.

Escalar la adopción de MCP: nuestra arquitectura de referencia para implementaciones empresariales de MCP

Compartimos la estrategia interna de Cloudflare para gestionar MCP mediante Access, AI Gateway y portales de servidor MCP. También lanzamos Code Mode para reducir drásticamente los costos de tokens y recomendar nuevas reglas de detección de Shadow MCP en Cloudflare Gateway.

Herramientas del Agente

Un agente competente debe ser capaz de pensar, recordar, comunicarse y ver. Esto significa contar con los modelos adecuados, con acceso a las herramientas adecuadas y el contexto apropiado para la tarea en cuestión. Esta semana presentamos las primitivas (inferencia, búsqueda, memoria, voz, correo electrónico y un navegador) que convierten a un agente en algo que realmente permite realizar el trabajo.

La plataforma de IA de Cloudflare: una capa de inferencia diseñada para agentes 

Estamos transformando Cloudflare en una capa de inferencia unificada para agentes, que permite a los desarrolladores acceder a modelos de más de 14 proveedores. Entre las novedades se incluye una integración de Workers para ejecutar modelos de terceros y un catálogo ampliado con modelos multimodales.

Construir la base para ejecutar modelos lingüísticos muy grandes

Creamos una pila tecnológica personalizada para ejecutar modelos lingüísticos de gran tamaño y rápidos en la infraestructura de Cloudflare. Este artículo analiza las compensaciones de ingeniería y las optimizaciones técnicas necesarias para que la inferencia de IA de alto rendimiento sea accesible.

Unweight: cómo logramos comprimir un LLM en un 22 % sin perder la calidad

La ejecución de grandes LLM en la red de Cloudflare nos exige optimizar el uso del ancho de banda de memoria de las GPU con mayor inteligencia y eficiencia. Por eso hemos desarrollado Unweight, un sistema de compresión sin pérdida aplicado durante la inferencia que logra reducir hasta un 22 % la huella del modelo, lo que nos permite ofrecer inferencias más rápidas y económicas que nunca. 

Agentes que recuerdan: presentamos Agent Memory

Cloudflare Agent Memory es un servicio gestionado que ofrece a los agentes de IA una memoria persistente, permitiéndoles recordar lo importante, olvidar lo irrelevante y volverse más inteligentes con el tiempo.

AI Search: la primitiva de búsqueda para tus agentes

AI Search es la primitiva de búsqueda para tus agentes. Crea instancias de forma dinámica, carga archivos y realiza búsquedas entre instancias con recuperación híbrida y refuerzo de relevancia. Solo tienes que crear una instancia de búsqueda, cargar y buscar.

Browser Run: brinda un navegador a tus agentes

Browser Rendering es ahora Browser Run, con Live View, Human in the Loop, acceso a CDP, grabaciones de sesiones y límites de concurrencia 4 veces mayores para los agentes de IA.
